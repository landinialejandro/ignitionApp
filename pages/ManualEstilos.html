SCSS Manual

Introduction

This manual explains the structure and usage of the SCSS files in this project. It includes detailed descriptions of each category and practical examples of their implementation in HTML.

File Structure

The SCSS files are organized into the following categories:

  • Base: Global styles, typography, resets, and variables.
  • Layout: Defines the structure of the application, including headers, footers, and sidebars.
  • Components: Modular and reusable UI elements like buttons, cards, and modals.
  • Utils: Helper classes and utility styles for spacing, responsiveness, and animations.
  • Themes: Style definitions for dark and light modes.

Base

The base folder includes foundational styles like typography, resets, and global settings.

Example: Typography

The file _typography.scss contains basic text styles for headings, paragraphs, and other textual elements.

Heading 1

This is a paragraph styled using base typography.

<h1>Heading 1</h1>
<p>This is a paragraph styled using base typography.</p>

Layout

The layout folder defines the overall structure of the app, including headers, footers, and sidebars.

Example: Header

The file _header.scss styles the application's header.

This is the header section
<header class="header">This is the header section</header>

Components

The components folder includes reusable UI elements like buttons, cards, and modals.

Example: Button

The file _button.scss defines styles for buttons.

<button class="btn-primary">Click Me</button>

Utils

The utils folder provides helper classes for spacing, responsiveness, and animations.

Example: Spacing

Utility classes like .m-10 and .p-20 help manage margins and padding.

Content with margin and padding
<div class="m-10 p-20">Content with margin and padding</div>

Themes

The themes folder includes definitions for dark and light modes.

Example: Dark Theme

The file _dark.scss defines styles for the dark theme.

This is a dark-themed component. 
<div class="dark-theme">This is a dark-themed component</div>
